# 1. 模块化
1. 指将一个大的程序文件，拆分成许多小的文件，然后将小文件组合起来；
# 2. 模块化的优点
1. 防止命名冲突；
2. 代码复用；
3. 高维护性；
# 3. 模块化规范产品
1. ES6之前模块化规范有：
   1. CommonJS ==> NodeJS/Browserify;
   2. AMD ==> requireJS;
   3. CMD ==> seaJS;
# 4. ES6模块化语法(案例20)
1. 模块功能主要由两个命令构成：export/import;
2. export：用于规定模块的对外接口:3种暴露方式
   1. 分别暴露：在暴露数据前面加export；
   2. 统一暴露：使用对象的写法简化数据；
   3. 默认暴露：暴露数据可以是任意类型；
3. import：用于输入其他模块提供的功能：3种导入形式
   1. 通用方式；
   2. 结构赋值形式：
   3. 简便形式：只针对默认暴露；
